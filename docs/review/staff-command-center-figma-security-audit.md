## What Was Found

- The Staff Command Center header still looked more like a hand-built approximation than the exported Figma shell.
- `Campaigns`, `Proof / UGC`, `Best Practices`, and `Admin` were visible as normal navigation even though those views redirect into the launch lane.
- `SOPs` linked to `/admin/sop-library`, which is not a shipped production route and returns `404`.
- The staff screen was using a simplified portfolio view instead of the exported Figma portfolio table and chapter drawer rhythm.
- User-generated media/UGC did not have an explicit shared allowlist and rejection utility for future embed work.

## Security Posture

- User-generated content should remain disabled for production until the full proof/upload/moderation workflow is approved.
- Arbitrary user-supplied embed code is blocked by policy and now has tests.
- Safe future embeds should be generated by the app from URLs only.
- Approved embed domains currently include Instagram, LinkedIn, Facebook, Loom, YouTube, youtu.be, and Vimeo.
- Unknown domains should render as plain links, not embeds.
- Uploads remain a future approved path; metadata validation now rejects invalid file types, mismatched extensions, and oversized files.
- Supabase RLS and hosted production policy proof are separate launch gates and were not changed in this UI/security repair pass.
